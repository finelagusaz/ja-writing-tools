# proofread-ja

日本語テキストの校正を行う Claude Code スキルです。

## 機能

- 誤字脱字の検出と修正提案
- 表記揺れのチェックと統一
- 文体の一貫性確認
- ら抜き言葉・さ入れ言葉の検出
- 重複表現のチェック

## 使い方

Claude Code で以下のように依頼してください：

```
/proofread-ja
```

または自然言語で：

- 「この文章を校正して」
- 「誤字脱字をチェックして」
- 「表記の揺れを確認して」

## 対応する文書タイプ

| タイプ | 説明 |
|---|---|
| 小説・エッセイ | 文体の一貫性を重視、意図的な表現を尊重 |
| 技術文書 | 用語の統一、正確さを重視 |
| ブログ・記事 | 読みやすさを重視 |
| ビジネス文書 | 敬語の正確さを重視 |

## ファイル構成

```
proofread-ja/
├── SKILL.md                    # スキル定義（ワークフロー、判断基準）
├── README.md                   # このファイル
└── references/
    ├── common-errors.md        # よくある誤字脱字パターン
    ├── consistency-rules.md    # 表記統一ルール
    └── custom-terms.md         # ユーザー固有の用語集（編集可能）
```

## カスタマイズ

`references/custom-terms.md` を編集することで、プロジェクト固有の用語や表記ルールを登録できます。

### 登録できる項目

- 固有名詞・製品名
- 専門用語
- 許容する表現（通常は誤りとされるが意図的に使用する表現）
- 禁止表現
- 文体設定（です・ます / だ・である）

## 校正結果の見方

校正結果は確信度別に分類されます：

| 確信度 | 説明 |
|---|---|
| 高 | 明らかな誤り。修正を強く推奨 |
| 中 | 表記揺れなど。統一方針の選択が必要 |
| 低 | 意図的な可能性あり。確認を推奨 |

## リファレンス

### common-errors.md

400以上の誤りパターンを収録：

- 同音異義語（以外/意外、作る/造る/創る など）
- 送り仮名の誤り
- ら抜き言葉・さ入れ言葉・れ足す言葉
- 重複表現（頭痛が痛い、まず最初に など）
- 慣用句の誤用
- 敬語の誤り

### consistency-rules.md

表記統一の判断基準：

- 漢字/ひらがな選択
- カタカナ長音符（サーバー/サーバ）
- 全角/半角（数字、英字、記号）
- 句読点（。、 vs ．，）
- 文体（敬体/常体）
- 括弧の使い分け
