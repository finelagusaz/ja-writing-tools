# 表記統一ルール

## 送り仮名の統一

### 基本ルール

送り仮名は「送り仮名の付け方」（内閣告示）に準拠する。

### 揺れやすい語

| 本則 | 許容 | 備考 |
|---|---|---|
| 行う | 行なう | 本則推奨 |
| 表す | 表わす | 本則推奨 |
| 現れる | 現われる | 本則推奨 |
| 断る | 断わる | 本則推奨 |
| 著す | 著わす | 本則推奨 |
| 終わる | 終る | 本則推奨 |
| 変わる | 変る | 本則推奨 |

### 複合動詞の送り仮名

| 正 | 誤 | 解説 |
|---|---|---|
| 書き込む | 書きこむ | 両方の動詞に送り仮名を付ける |
| 立ち上がる | 立ちあがる | 同上 |
| 取り消す | 取りけす | 同上 |
| 打ち合わせ | 打合せ | 名詞化しても送り仮名を維持（慣例で省略可） |

### 名詞化の送り仮名

| 動詞 | 名詞形 | 備考 |
|---|---|---|
| 受け付ける | 受付 / 受け付け | 名詞は省略可（統一すること） |
| 取り扱う | 取扱 / 取り扱い | 同上 |
| 申し込む | 申込 / 申し込み | 同上 |
| 引き継ぐ | 引継 / 引き継ぎ | 同上 |

## 漢字/ひらがな選択

### 基本方針

- 小説: 柔らかい印象を優先 → ひらがな多め
- 技術文書: 正確さを優先 → 漢字多め

### 判断が分かれる語

| 語 | 小説推奨 | 技術文書推奨 | 備考 |
|---|---|---|---|
| 事/こと | こと | 事 | 名詞化は「こと」優先 |
| 時/とき | とき | 時 | |
| 物/もの | もの | 物 | 抽象概念は「もの」 |
| 等/など | など | 等 | |
| 所/ところ | ところ | 所 | 形式名詞は「ところ」 |
| 訳/わけ | わけ | わけ | 両方ひらがな推奨 |
| 様/よう | よう | よう | 形式名詞は「よう」 |
| 為/ため | ため | ため | 両方ひらがな推奨 |
| 程/ほど | ほど | 程 | 程度の意味は「ほど」 |
| 通り/とおり | とおり | 通り | 形式名詞は「とおり」 |
| 故/ゆえ | ゆえ | 故 | 文語的な文書のみ漢字 |
| 毎/ごと | ごと | 毎 | 「〜するたびに」は「ごと」 |

### 動詞の漢字/ひらがな

| 漢字表記 | ひらがな表記 | 使い分け |
|---|---|---|
| 言う | いう | 「〜という」は形式動詞なので「いう」 |
| 有る/在る | ある | 補助動詞は「ある」（〜してある） |
| 居る | いる | 補助動詞は「いる」（〜している） |
| 見る | みる | 補助動詞は「みる」（〜してみる） |
| 行く | いく | 補助動詞は「いく」（〜していく） |
| 来る | くる | 補助動詞は「くる」（〜してくる） |
| 貰う | もらう | 補助動詞は「もらう」（〜してもらう） |
| 遣る | やる | 「やる」が一般的 |
| 出来る | できる | 可能の意味は「できる」 |

## 単位の表記

### 数値と単位の間

| 文書タイプ | 推奨 | 例 |
|---|---|---|
| 技術文書 | スペースなし | 100km、50kg |
| 学術論文 | 半角スペース | 100 km、50 kg（SI単位系準拠） |

### 単位の表記形式

| 種類 | 推奨 | 避ける | 備考 |
|---|---|---|---|
| SI単位 | 半角英字 | 全角・カタカナ | m、kg、s |
| 派生単位 | 半角英字 | 機種依存文字 | km（㎞は避ける） |

| 時間 | 統一する | 混在 | 「時間」「h」「hr」いずれか |

### 主な単位表記

| SI記号 | カタカナ | 漢字 | 使用場面 |
|---|---|---|---|
| m | メートル | 米 | 技術文書はm、一般文書はメートル可 |
| km | キロメートル | 粁 | kmが一般的、漢字は避ける |
| kg | キログラム | 瓩 | kgが一般的、漢字は避ける |
| L / l | リットル | 立 | Lを推奨（小文字lは1と紛らわしい） |
| ℃ | 度 | 度 | ℃または「〜度」 |

## リスト・箇条書きの記号

### 記号の種類と用途

| 記号 | 用途 | 備考 |
|---|---|---|
| ・（中黒） | 一般的な箇条書き | 最も汎用的 |
| ●（黒丸） | 強調したい箇条書き | プレゼン資料向き |
| ○（白丸） | 副項目、補足 | ●の下位項目として |
| ■（黒四角） | 見出し的な箇条書き | 強い強調 |
| □（白四角） | チェックリスト（未完了） | タスク管理 |
| ☑（チェック済み） | チェックリスト（完了） | タスク管理 |
| -（ハイフン） | Markdown形式 | 技術文書、README |
| *（アスタリスク） | Markdown形式 | 技術文書 |
| 1. 2. 3. | 順序のあるリスト | 手順説明 |
| (1) (2) (3) | 公文書形式 | 法律文書、契約書 |
| ① ② ③ | 参照番号 | 機種依存に注意 |

### 階層構造の表現

```
■ 大項目
  ● 中項目
    ○ 小項目
    ○ 小項目
  ● 中項目

または

1. 大項目
   1.1 中項目
   1.2 中項目
      1.2.1 小項目
```

### 統一ルール

- 同一文書内では記号を統一する
- 階層は3段階までを推奨（深すぎると読みにくい）
- 箇条書きの文末は「。」を付けない形式と付ける形式があるが、統一する

## ダッシュ・ハイフンの使い分け

### 記号の種類

| 記号 | 名称 | Unicode | 用途 |
|---|---|---|---|
| - | ハイフンマイナス | U+002D | 複合語、マイナス記号 |
| ‐ | ハイフン | U+2010 | 複合語（正式） |
| – | enダッシュ | U+2013 | 範囲（1–10）、欧文 |
| — | emダッシュ | U+2014 | 挿入句、欧文 |
| ― | ホリゾンタルバー | U+2015 | 日本語の傍線 |
| ー | 長音符 | U+30FC | カタカナの長音 |
| 〜 | 波ダッシュ | U+301C | 範囲（1〜10） |
| ～ | 全角チルダ | U+FF5E | 波ダッシュの代用（避ける） |

### 日本語文書での使い分け

| 用途 | 推奨 | 例 |
|---|---|---|
| 範囲の表現 | 〜（波ダッシュ） | 10〜20、月〜金 |
| カタカナの長音 | ー（長音符） | サーバー、コンピューター |
| 複合語の連結 | -（ハイフン） | Eメール、T-シャツ |
| 電話番号 | -（ハイフン） | 03-1234-5678 |
| 日付区切り | -（ハイフン） | 2024-01-01 |
| 言い換え・挿入 | ―（ホリゾンタルバー） | 彼女は―信じられないことに―勝った |

### 注意点

- 波ダッシュ（〜）と全角チルダ（～）は別の文字。波ダッシュを推奨
- 長音符（ー）を範囲の表現に使わない（×10ー20）
- enダッシュ・emダッシュは欧文向け。日本語文書では使用を避ける

## 句読点

### 種類の選択

| 種類 | 用途 | 例 |
|---|---|---|
| 。、（テン・マル） | 一般的な日本語文書 | 私は、学生です。 |
| ．，（ピリオド・カンマ） | 学術論文、公文書 | 私は，学生です． |
| 。，（混合） | 避ける | （なし） |

### 縦書き/横書きの違い

| 項目 | 横書き | 縦書き |
|---|---|---|
| 句読点 | 、。 | 、。（位置が異なる） |
| 数字 | 半角アラビア数字 | 漢数字推奨 |
| 疑問符・感嘆符 | ？！ | 縦向きまたは省略 |
| ダッシュ | ― | ｜（縦線） |
| 括弧の向き | （） | 縦向き括弧 |

### 句読点の打ち方

| ルール | 良い例 | 悪い例 |
|---|---|---|
| 主語の後（長い場合） | 昨日駅で会った友人は、元気だった。 | 昨日駅で会った友人は元気だった。 |
| 接続詞の後 | しかし、彼は来なかった。 | しかし彼は来なかった。 |
| 並列要素の区切り | りんご、みかん、バナナ | りんごみかんバナナ |
| 文節ごとには不要 | 私は昨日、駅で友人に会った。 | 私は、昨日、駅で、友人に、会った。 |

### 全角/半角スペース

| 場面 | ルール | 例 |
|---|---|---|
| 日本語と英数字の間 | スペースなし or 半角 | 統一する |
| 英単語間 | 半角スペース | Hello World |
| 日本語文中 | スペースなし | 今日は晴れです |

### 技術文書でのコードと本文

| 場面 | ルール | 例 |
|---|---|---|

## 括弧の使い分け

| 括弧 | 用途 | 例 |
|---|---|---|
| 「」 | 引用、発言、強調 | 彼は「行く」と言った。 |
| 『』 | 書名、作品名、「」内の引用 | 『源氏物語』を読んだ。 |
| （） | 補足説明、読み仮名 | 東京（とうきょう） |
| 【】 | 見出し、ラベル、強調 | 【重要】お知らせ |
| 〈〉 | 引用内の引用、やや弱い強調 | 〈注意〉してください |
| 《》 | 二重山括弧、作品名 | 《モナリザ》 |
| ［］ | 注釈、編集者追記 | ［原文ママ］ |

### 引用符のスタイル

| 括弧 | 用途 | 例 |
|---|---|---|
| 「」 | 日本語の引用・発言 | 彼は「はい」と答えた。 |
| 『』 | 「」内の引用、書名 | 「『源氏物語』を読んだ」と彼は言った。 |
| ""（ダブルクォート） | 英文の引用 | He said "Hello". |
| ''（シングルクォート） | ""内の引用、英文 | He said "I read 'War and Peace'". |

### 日本語文中の英語引用

| パターン | 例 | 備考 |
|---|---|---|
| 日本語括弧で統一 | 彼は「Hello」と言った。 | 日本語文中は「」推奨 |
| 英語括弧を維持 | 英語で"Hello"と書く。 | 英語表現を強調する場合 |

### 括弧と句読点の位置

| パターン | 例 | 備考 |
|---|---|---|
| 句点は括弧の外 | 〜です（補足）。 | 一般的 |
| 括弧内が独立文 | （これは補足です。） | 括弧内にも句点 |
| 発言の終わり | 「行きます」 | 括弧内に句点なし |
| 発言が文末 | 彼は「行く」と言った。 | 括弧外に句点 |

### 括弧のネスト

| 深さ | 推奨 | 備考 |
|---|---|---|
| 1段階 | 「〜」 | 基本 |
| 2段階 | 「『〜』」 | 許容 |
| 3段階以上 | 避ける | 文を分割するか、別の表現を検討 |

悪い例：

```
彼は「『「本当に？」と彼女は言った』と書いてあった」と言った。
```

改善例：

```
彼は、彼女の「本当に？」という発言が引用されていたと言った。
```

## 固有名詞の表記

### 基本方針

- 公式表記に従う（企業・製品の表記は公式サイトを確認）
- 表記が変更された場合は最新の公式表記を使用
- 文書内で統一する

### よくある表記の揺れ

| 公式表記 | 誤りやすい表記 | 備考 |
|---|---|---|
| iPhone | Iphone、i-phone | 大文字小文字に注意 |
| macOS | MacOS、MACOS | 公式表記を確認 |
| JavaScript | Javascript、JAVASCRIPT | 正式名称を使用 |
| GitHub | Github、GITHUB | 大文字小文字に注意 |
| YouTube | Youtube、YOUTUBE | 同上 |
| Wi-Fi | WiFi、Wifi、WIFI | ハイフンあり |
| Bluetooth | BlueTooth、BLUETOOTH | 大文字小文字に注意 |

### 日本語表記

| 種類 | 方針 | 例 |
|---|---|---|
| 企業名 | 公式表記 | ソフトバンク（×ソフト・バンク） |
| 地名 | 常用漢字＋公式表記 | 東京都、さいたま市 |
| 人名 | 本人の希望・公式表記 | 統一する |
| 作品名 | 原題・公式タイトル | 『』で括る |

## 注釈・脚注の形式

### 注釈記号の種類

| 記号 | 用途 | 例 |
|---|---|---|
| ※ | 補足説明、注意事項 | ※本サービスは〜 |
| *（アスタリスク） | 脚注参照（欧文式） | 詳細は下記*を参照 |
| †（ダガー） | 脚注参照（学術） | 研究結果†によると |
| 注1、注2 | 連番の注釈 | 〜である（注1）。 |
| [1]、[2] | 参考文献の参照 | 先行研究[1]では |
| (1)、(2) | 補足説明の連番 | 条件(1)を満たす場合 |

### 注釈の配置

| 形式 | 用途 | 例 |
|---|---|---|
| 文末脚注 | 長い補足説明 | ページ下部または文書末 |
| 括弧内注釈 | 短い補足 | 〜である（詳細は後述）。 |
| ※による注記 | 注意事項 | ※この機能は〜 |
| インライン | 読み仮名、略語展開 | AI（人工知能） |

### 統一ルール

- 同一文書内で注釈形式を統一する
- 番号付き注釈は本文の出現順に連番を振る
- 注釈が多すぎる場合は本文に組み込むことを検討

## 文体の一貫性

### 敬体/常体

| 文体 | 特徴 | 用途 |
|---|---|---|
| 敬体（です・ます） | 丁寧、読者との距離感 | ブログ、マニュアル、ビジネス文書 |
| 常体（だ・である） | 簡潔、断定的 | 論文、小説、報告書 |

### 混在のチェック

```
# 敬体→常体の混在（要修正）
これは重要な機能です。処理が高速だ。

# 修正例
これは重要な機能です。処理が高速です。
```

### 許容される混在

| パターン | 例 | 備考 |
|---|---|---|
| 引用内 | 「これは問題だ」と彼は言いました。 | 引用は元の文体を維持 |
| 箇条書き | ・処理が高速である / 本文は「です・ます」 | 箇条書きは常体も可 |
| 見出し | 【注意事項】/ 本文は「です・ます」 | 見出しは常体も可 |

## 一人称の統一

| 一人称 | 印象 | 用途 |
|---|---|---|
| 私（わたし） | フォーマル、中性 | ビジネス、論文 |
| 私（わたくし） | より丁寧 | 式典、公式文書 |
| 僕 | カジュアル、男性的 | ブログ、エッセイ |
| 俺 | くだけた、男性的 | 小説の一人称 |
| 自分 | 中性、やや硬い | 報告書 |
| 筆者 | 客観的 | 論文、解説 |
| 著者 | 客観的 | 書籍 |

## 接続詞の統一

### 逆接

| 表現 | フォーマル度 | 備考 |
|---|---|---|
| しかし | 高 | 論文、ビジネス |
| だが | 中 | 常体の文章 |
| けれども | 中 | 敬体の文章 |
| でも | 低 | 口語的 |
| ただ | 中 | 補足的な逆接 |

### 順接

| 表現 | フォーマル度 | 備考 |
|---|---|---|
| したがって | 高 | 論理的帰結 |
| そのため | 高 | 因果関係 |
| だから | 中 | 常体向き |
| ですから | 中 | 敬体向き |
| なので | 低 | 文頭では避ける |

### 添加

| 表現 | フォーマル度 | 備考 |
|---|---|---|
| また | 高 | 汎用的 |
| さらに | 高 | 追加・強調 |
| 加えて | 高 | フォーマル |
| そして | 中 | 時系列にも |
| それに | 低 | 口語的 |
